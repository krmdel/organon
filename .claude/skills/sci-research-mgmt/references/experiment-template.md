# Experiment Log Template

## File Location

`research/experiments/YYYY-MM-DD_{slug}.md`

- `YYYY-MM-DD` is the date the experiment log is promoted (not the original note date)
- `{slug}` is derived from the entry title: lowercase, hyphens, max 50 characters

---

## Template

```markdown
---
title: {Descriptive experiment title}
promoted_from: research/notes/{date}.md
promoted_at: {ISO 8601 timestamp}
status: planning
linked_project: {project-slug or empty}
---

# {Title}

## Hypothesis
{Copied from note, or filled manually, or generated by sci-hypothesis}

## Method
{To be filled -- or pipe to sci-hypothesis design mode}

## Results
{Pending}

## Notes
{Original context from the promoted note entry.}
```

---

## Promotion Flow

1. Find entries tagged `#experiment` in the specified day's note file (`research/notes/{date}.md`)
2. Copy entry content to new file using template above
3. Generate slug from entry title (lowercase, hyphens, max 50 chars):
   - Example: "Unexpected folding pattern" → `unexpected-folding-pattern`
4. Ask: "Pipe to sci-hypothesis for full experiment design? [y/N]"
5. If yes: pass hypothesis text to sci-hypothesis generate mode

If multiple `#experiment` entries exist in the same note file, prompt the scientist to select which entry to promote. One promotion per invocation.

---

## Status Values

`planning | active | complete | abandoned`

| Status | Meaning |
|--------|---------|
| `planning` | Initial state at promotion — hypothesis captured, method not finalized |
| `active` | Experiment is running, data collection in progress |
| `complete` | Experiment finished, results recorded |
| `abandoned` | Experiment stopped before completion (record reason in Notes) |

---

## Slug Generation Rules

- Convert title to lowercase
- Replace spaces and non-alphanumeric characters with hyphens
- Remove consecutive hyphens
- Truncate to 50 characters maximum
- Example: `Temperature gradient experiment` → `temperature-gradient-experiment`

---

## sci-hypothesis Integration

When promoting an entry, after creating the experiment log, Claude offers to pipe the hypothesis to `sci-hypothesis` generate mode. This populates:

- **Hypothesis**: Formalizes the note entry as a testable hypothesis with null/alternative statements
- **Method**: Generates experiment design with sample size calculation and statistical test plan

Promotion success message: `Promoted to research/experiments/{date}_{slug}.md. Fill in the Method and Results sections when ready.`

Promotion with hypothesis pipe: `Promoted to experiment log and piped to sci-hypothesis for experiment design.`
