# Literature Monitor Pipeline

## Trigger
Manual invocation or scheduled via cron job (science-paper-alerts).

## Input
- `topic`: Research topic or query string (required)
- `max_results`: Number of papers per search (default: 5)
- `source`: Search sources -- `"all"` for PubMed + arXiv + OpenAlex + Semantic Scholar (default: `"all"`)

## Steps
1. **Search**: Use sci-literature-research (search mode) for the given topic with source "all" and max_results setting. Filter to papers from the last 7 days if running as a periodic check.
2. **Summarize**: For top 3 results by citation count, use sci-literature-research (summarize mode) to generate structured summaries (key findings, methods, limitations).
3. **Alert**: Compile search results and summaries into a single alert file at `research/alerts/lit-monitor_{YYYY-MM-DD}.md`.

## Output
`research/alerts/lit-monitor_{YYYY-MM-DD}.md`

Output format:

```markdown

## Summary
Found {N} new papers on "{topic}".

## Top Papers

### 1. {Title}
- Authors: {authors}
- Journal: {journal}
- DOI: {doi}
- Key findings: {summary}

### 2. {Title}
- Authors: {authors}
- Journal: {journal}
- DOI: {doi}
- Key findings: {summary}

### 3. {Title}
- Authors: {authors}
- Journal: {journal}
- DOI: {doi}
- Key findings: {summary}

## All Results
| # | Title | Authors | Year | Citations | Source |
|---|-------|---------|------|-----------|--------|
| 1 | {title} | {first author} et al. | {year} | {count} | {source} |

## Pipeline Status
- Step 1: Search -- complete | failed
- Step 2: Summarize -- complete | failed | skipped
- Step 3: Alert -- complete | failed

---
*Generated by literature-monitor pipeline at {ISO 8601 timestamp}*
```

## Error Handling
- If search fails: retry once. If still fails, log `Search failed for topic: {topic}` and skip summarize step. Still produce output file with failure note.
- If summarize fails for a paper: skip that paper, continue with remaining papers. Log which summaries were skipped.
- Always produce output file, even if partial. Include Pipeline Status section showing which steps succeeded/failed.
- Skipped steps (due to upstream failure) are marked `skipped` in Pipeline Status.
