# Data Watch Pipeline

## Trigger
Manual invocation or scheduled via cron job (science-data-monitor).

## Input
- `dataset_path`: Path to the dataset file to monitor (required)
- `checks`: What to check -- `"profile"` (shape/types/stats), `"drift"` (compare to baseline), or `"both"` (default: `"profile"`)

## Steps
1. **Load**: Use sci-data-analysis (load mode) to read the dataset and get current shape, column types, and summary statistics.
2. **Analyze**: Use sci-data-analysis (profile mode) to generate descriptive statistics. If `checks="drift"`, compare current stats to the last saved profile in `research/alerts/`.
3. **Flag**: If any significant changes detected (new columns, missing data increase >5%, row count change >10%), flag as `**CHANGED**`. Otherwise flag as `**STABLE**`.

## Output
`research/alerts/data-watch_{YYYY-MM-DD}.md`

Output format:

```markdown
# Data Watch - {YYYY-MM-DD}

## Summary
Dataset: `{dataset_path}`
Status: **STABLE** | **CHANGED**
Rows: {count} | Columns: {count}

## Changes Detected
- {change description}

(or: "No significant changes detected." if STABLE)

## Current Profile

| Column | Type | Non-null | Mean | Std | Min | Max |
|--------|------|----------|------|-----|-----|-----|
| {col} | {type} | {count} | {val} | {val} | {val} | {val} |

## Pipeline Status
- Step 1: Load -- complete | failed
- Step 2: Analyze -- complete | failed
- Step 3: Flag -- complete | failed

---
*Generated by data-watch pipeline at {ISO 8601 timestamp}*
```

## Error Handling
- If load fails: retry once. If still fails, log `Cannot load dataset: {path}` and exit pipeline. Still produce output file with failure note.
- If analyze fails: retry once. If still fails, log `Analysis failed` but still produce output with load results (shape and column types only).
- Always include Pipeline Status section in the output file.
- Flag step only runs if both Load and Analyze succeeded. Otherwise mark as `skipped`.

## Change Detection Thresholds
- **New or removed columns**: Always flag as CHANGED (structural change)
- **Missing data increase**: Flag if any column's null count increases by >5% vs baseline
- **Row count change**: Flag if row count changes by >10% vs baseline
- **Baseline source**: Most recent prior `data-watch_*.md` file in `research/alerts/` for the same dataset path. If no baseline exists, skip drift check and run profile only.
